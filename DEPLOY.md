# Деплой ride_bot: пошаговая инструкция

Бот работает через long polling (не вебхук), поэтому нужен только
долгоживущий процесс с доступом в интернет — публичный URL и HTTPS не нужны.
Ниже — все аккаунты и шаги от нуля до работающего бота в проде.

Понадобится:
1. Telegram-бот (BotFather) — бесплатно, без карты.
2. Проект Firebase/Firestore — бесплатно на тарифе Spark, без карты.
3. Хостинг для процесса — в этой инструкции используется **ваш Raspberry Pi с
   Home Assistant OS (HAOS)**, бот ставится как локальный HA-аддон. Полностью
   бесплатно, никакого стороннего облака. (Render/Fly.io/Railway по состоянию
   на 2026 год не дают реально бесплатного always-on тарифа для воркера —
   упомянуты в конце документа только как платная альтернатива.)

## 1. Создать бота в Telegram (BotFather)

1. Откройте в Telegram чат [@BotFather](https://t.me/BotFather).
2. Отправьте `/newbot`.
3. Введите имя бота (отображается в чатах) — например `Ride Planner`.
4. Введите username бота — обязательно заканчивается на `bot`, например `our_ride_planner_bot`.
5. BotFather пришлёт **токен** вида `123456789:AAExampleTokenXXXXXXXXXXXXXXXXXXXXX`.
   Это значение — `BOT_TOKEN`. Никому не показывайте, это полный доступ к боту.
6. (Необязательно, но рекомендуется) отправьте `/setprivacy` → выберите бота →
   **Disable**. Это позволяет боту видеть все сообщения в группе, а не только
   команды, адресованные ему явно — не обязательно для команд (`/suggest` и
   т.д. работают всегда), но полезно на будущее.
7. Добавьте бота в вашу группу (обычным способом — «Добавить участника» →
   найти по username бота).
8. Сделайте бота **администратором группы**: настройки группы →
   «Администраторы» → «Добавить администратора» → выбрать бота. Права можно
   оставить дефолтные — этого достаточно, чтобы бот мог отправлять опросы,
   закрывать их (`stopPoll`) и читать список админов.

### Как узнать `GROUP_CHAT_ID`

Это отрицательное число вида `-1001234567890`. Самый простой способ:

1. Временно добавьте в группу бота [@RawDataBot](https://t.me/RawDataBot) (или
   любой похожий "id echo" бот), отправьте туда любое сообщение — он ответит
   JSON-ом с полем `chat.id`. После этого можно удалить его из группы.
2. Либо, если уже подняли ride_bot локально (см. ниже) — временно добавьте в
   `src/index.ts` вывод `console.log(ctx.chat.id)` в любой хендлер, отправьте
   `/suggest test` в группу, посмотрите в консоль, потом уберите отладочный вывод.

## 2. Создать проект Firestore (Firebase)

1. Перейдите на [console.firebase.google.com](https://console.firebase.google.com/)
   и войдите с Google-аккаунтом (тем же, что и `nea.dima@gmail.com`, если
   удобно — не принципиально, главное свой Google-аккаунт).
2. **Add project** → введите название, например `ride-bot` → далее можно
   отключить Google Analytics (не нужен) → **Create project**.
3. В левом меню: **Build → Firestore Database** → **Create database**.
   - Выберите регион (любой ближайший, например `eur3 (europe-west)`).
   - Режим: **Start in production mode** (правила по умолчанию блокируют
     чтение/запись извне — это ок, бот ходит в Firestore через Admin SDK,
     который эти правила игнорирует).
4. Это бесплатный тариф **Spark** — для нагрузки этого бота (одна группа,
   несколько десятков документов) он не будет превышен. Карта не требуется.

### Получить сервисный ключ (service account)

1. В консоли Firebase: значок шестерёнки рядом с "Project Overview" →
   **Project settings**.
2. Вкладка **Service accounts** → кнопка **Generate new private key** →
   подтвердить. Скачается файл `<project-id>-firebase-adminsdk-xxxxx.json`.
3. **Храните этот файл в секрете** — он даёт полный доступ к Firestore проекта.
   Не коммитьте его в git.
4. Из этого файла понадобятся:
   - `project_id` → переменная `FIREBASE_PROJECT_ID`
   - Всё содержимое файла целиком (как есть, в одну строку) →
     переменная `FIREBASE_SERVICE_ACCOUNT_JSON`

   Чтобы превратить JSON-файл в одну строку для `.env`, можно выполнить:
   ```bash
   node -e "console.log(JSON.stringify(require('/путь/к/ключу.json')))"
   ```
   Результат — вставить как значение `FIREBASE_SERVICE_ACCOUNT_JSON` в `.env`
   (для локального запуска) или в переменные окружения хостинга (для прода).

## 3. Локальный запуск (проверить перед деплоем)

1. Установите Node.js 20+, если ещё нет.
2. В папке проекта:
   ```bash
   npm install
   cp .env.example .env
   ```
3. Заполните `.env`:
   ```
   BOT_TOKEN=<токен из BotFather>
   GROUP_CHAT_ID=<id вашей группы, отрицательное число>
   FIREBASE_PROJECT_ID=<project_id из service account>
   FIREBASE_SERVICE_ACCOUNT_JSON=<весь JSON-ключ одной строкой>
   DEFAULT_TIMEZONE=Europe/Moscow
   ```
4. Запуск:
   ```bash
   npm run dev
   ```
5. Проверьте в группе: `/suggest Куда-нибудь`, `/list`, `/create_poll` —
   должно всё отработать. Если Firestore ругается на отсутствующий составной
   индекс (`FAILED_PRECONDITION`) — в тексте ошибки будет прямая ссылка,
   перейдите по ней один раз, подтвердите создание индекса в консоли Firebase
   (занимает 1-2 минуты), повторите команду.

## 4. Деплой на Raspberry Pi (Home Assistant OS) как локальный аддон

HAOS — закрытая ОС без обычного shell-доступа: сервисы запускаются через
Supervisor как Docker-контейнеры ("аддоны"). Бот в этом репозитории уже
упакован как такой аддон — в корне проекта лежат `config.yaml`, `build.yaml`,
`Dockerfile`, `run.sh`, `.dockerignore`. Ничего писать не нужно, только
скопировать проект на Pi и поставить через UI.

### 4.1. Открыть доступ к папке `/addons` на Pi

1. В Home Assistant: **Настройки → Аддоны → Магазин аддонов** → в правом
   верхнем углу (⋮) → **Repositories**, либо сразу ищите официальный аддон
   **"Samba share"** в магазине → **Install**.
2. Откройте аддон **Samba share** → вкладка **Configuration** — можно оставить
   значения по умолчанию (либо задать `username`/`password`, если хотите
   защитить паством) → **Start**.
3. На Mac: Finder → **Go → Connect to Server…** (⌘K) → введите
   `smb://homeassistant.local/addons` (или `smb://<IP вашего Pi>/addons`) →
   подключитесь (Guest, либо логин/пароль из шага 2). Откроется сетевая папка
   `addons`, смонтированная с Pi.

### 4.2. Скопировать проект в `/addons/ride_bot`

1. В смонтированной папке `addons` создайте подпапку `ride_bot`.
2. Скопируйте туда **весь** проект `ride_bot` (папку `src/`, `package.json`,
   `package-lock.json`, `tsconfig.json`, `config.yaml`, `build.yaml`,
   `Dockerfile`, `run.sh`, `.dockerignore`) — **кроме** `node_modules/`,
   `dist/` и `.env` (они не нужны и `.env` содержит секреты, которые сюда
   передавать не надо — конфигурация задаётся через UI аддона, см. ниже).

### 4.3. Установить аддон

1. Вернитесь в Home Assistant: **Настройки → Аддоны → Магазин аддонов** →
   (⋮) → **Check for updates** (или просто обновите страницу). В самом низу
   страницы магазина появится секция **Local add-ons** с карточкой **Ride Bot**.
2. Откройте её → **Install** (сборка Docker-образа на самом Pi — с `npm ci` и
   `npm run build`, может занять несколько минут на слабом Pi, это нормально).

### 4.4. Настроить переменные и запустить

1. После установки откройте вкладку **Configuration** аддона Ride Bot и
   заполните поля (это ровно те же значения, что описаны в разделах 1–2 этой
   инструкции):
   ```yaml
   bot_token: "<токен из BotFather>"
   group_chat_id: "-1001234567890"
   firebase_project_id: "<project_id из service account>"
   firebase_service_account_json: '<весь JSON-ключ одной строкой>'
   default_timezone: "Europe/Moscow"
   ```
   Сохраните (**Save**).
2. На вкладке **Info** включите тумблеры **Start on boot** и **Watchdog**
   (автозапуск при перезагрузке Pi и автоперезапуск, если процесс упадёт).
3. Нажмите **Start**. Вкладка **Log** должна показать `[bot] Started as @...`.
4. Проверьте в группе: `/suggest Куда-нибудь`, `/list`, `/create_poll` —
   так же, как в локальном тесте из раздела 3.

### Обновление бота в будущем

Если поменяете код (например, я внесу правки) — обновите файлы в
`/addons/ride_bot` тем же способом (шаг 4.2), затем в UI аддона: **Rebuild**
(не просто Restart — нужно пересобрать образ с новым кодом).

## 5. После деплоя — настройка бота в группе

1. Каждый, кто должен получать DM-уведомления о новых `/suggest` (обычно все
   реальные админы группы), должен один раз написать боту `/start` в личных
   сообщениях — иначе Telegram не даст боту написать первым.
2. Настройте расписание автосоздания опроса:
   ```
   /set_schedule friday 10:00 Europe/Moscow
   ```
3. Проверьте `/list`, добавьте несколько `/suggest`, запустите `/create_poll`
   вручную один раз, проверьте `/close_poll` с голосованием и подтверждением
   победителя.

## Итоговый список аккаунтов, которые понадобится создать

| Сервис | Зачем | Нужна ли карта |
|---|---|---|
| Telegram (BotFather) | сам бот и токен | нет |
| Google / Firebase | Firestore как база данных | нет (тариф Spark) |
| Home Assistant (уже есть) | хостинг долгоживущего процесса бота (локальный аддон) | нет |

## Приложение: облачные хостинги (платно, на случай отказа от Pi)

Если Raspberry Pi станет недоступен (переезд, поломка и т.п.) — по состоянию
на 2026 год ни один из "привычных" облачных хостингов не даёт реально
бесплатного always-on тарифа для background-воркера:

- **Render** — бесплатен только Web Service, который засыпает через 15 минут
  простоя (не годится для long-polling бота). Background Worker — от $7/мес.
- **Fly.io** — бесплатного тарифа больше нет, нужна карта, от ~$5/мес даже за
  самый маленький always-on процесс.
- **Railway** — стартовый кредит $5 на 30 дней, потом $1/мес бесплатных
  кредитов; расход этого бота (~$0.3–1/мес) близок к границе — может хватать,
  но не гарантированно навсегда бесплатно.
- **Oracle Cloud Free Tier** — единственный по-настоящему бесплатный навсегда
  вариант в облаке: постоянная ARM-VM (до 4 vCPU/24GB). Требует ручной
  настройки Linux-сервера + карту только для верификации личности (списаний
  не будет), плюс лёгкий heartbeat, чтобы Oracle не посчитал VM простаивающей
  и не удалил её через 7 дней бездействия.

Деплой на любой из них — обычный Node.js процесс (`npm install && npm run
build && npm start`), без Docker-обвязки из раздела 4 (та нужна только для
формата HAOS-аддона).
