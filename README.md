# ride_bot

Telegram-бот для одной группы друзей: участники предлагают маршруты/направления
поездок, админ группы запускает опрос (вручную или по расписанию), а выбранное
место исключается из будущих опросов, пока админ не вернёт его вручную.

## Стек

- Node.js + TypeScript
- [grammY](https://grammy.dev/) — Telegram Bot API, long polling
- Firestore (через `firebase-admin`) — хранилище данных
- `node-cron` — расписание автосоздания опроса
- `luxon` — валидация дня недели / времени / IANA-таймзоны

## Команды

| Команда | Кто | Где | Что делает |
|---|---|---|---|
| `/suggest <текст>` | любой участник | группа | добавить вариант маршрута |
| `/list` | админ | группа/ЛС | список активных вариантов |
| `/edit <номер> <текст>` | админ | группа/ЛС | изменить текст варианта |
| `/delete <номер>` | админ | группа/ЛС | удалить вариант навсегда (с подтверждением) |
| `/create_poll` | админ | группа/ЛС | создать опрос сейчас |
| `/set_schedule <день> <ЧЧ:ММ> <таймзона>` | админ | группа/ЛС | настроить автосоздание опроса |
| `/close_poll` | админ | группа | закрыть опрос, зафиксировать победителя |
| `/restore <номер>` | админ | группа/ЛС | вернуть исключённый вариант в пул |
| `/excluded` | админ | группа/ЛС | список исключённых (использованных) вариантов |
| `/set_reminder_time <ЧЧ:ММ> <таймзона>` | админ | группа/ЛС | настроить время ежедневной памятки про `/suggest` |
| `/set_reminder_text <текст>` | админ | группа/ЛС | заменить текст памятки на свой |
| `/get_reminder` | админ | группа/ЛС | показать текущее время и текст памятки |
| `/start` | любой | ЛС | зарегистрироваться на DM-уведомления (нужно админам) |

Админский статус проверяется live через Telegram (`getChatMember`) — это
реальные админы/создатель группы, не отдельный список.

## Ежедневная памятка

Если задать время через `/set_reminder_time`, бот раз в день будет присылать
в группу напоминание предложить вариант поездки (текст по умолчанию — см.
`DEFAULT_REMINDER_TEXT` в `src/constants.ts`). Пока время не задано —
напоминание не отправляется. Текст можно заменить своим через
`/set_reminder_text`, показывается он сразу с придуманным по умолчанию.

## Опрос

- Не анонимный (`is_anonymous: false`).
- Максимум **8 вариантов всего**: до 7 реальных предложений (по дате добавления)
  + обязательный вариант **«Мимокрокодил»**, который добавляется всегда последним.
  Если побеждает «Мимокрокодил» — в этот раз никуда не едем, ни одно предложение
  не исключается из пула.

**Важно:** Telegram не позволяет боту первым написать пользователю в ЛС.
Чтобы админ получал уведомления о новых `/suggest`, он должен один раз
написать боту `/start` в личных сообщениях.

## Локальный запуск

1. Создать бота через [@BotFather](https://t.me/BotFather), получить `BOT_TOKEN`.
2. Добавить бота в тестовую группу и сделать админом.
3. Получить `GROUP_CHAT_ID` (например, залогировав `ctx.chat.id` из любого апдейта).
4. Создать проект Firestore, сгенерировать ключ сервисного аккаунта (JSON).
5. Скопировать `.env.example` → `.env` и заполнить все переменные
   (`FIREBASE_SERVICE_ACCOUNT_JSON` — весь JSON-ключ одной строкой).
6. Установить зависимости и запустить:

```bash
npm install
npm run dev
```

## Деплой

Бот работает через long polling — публичный URL/вебхук не нужен, только
долгоживущий процесс.

Основной вариант — **Raspberry Pi с Home Assistant OS**, бот ставится как
локальный HA-аддон (файлы `config.yaml`, `build.yaml`, `Dockerfile`, `run.sh`
в корне проекта). Подробная пошаговая инструкция — в [DEPLOY.md](./DEPLOY.md).

Для обычного Node-хостинга (VPS, Oracle Cloud Free Tier и т.п., без HAOS):

```bash
npm run build
npm start
```

`Procfile` объявляет процесс как `worker` (background worker, без HTTP-порта).
Если конкретный хостинг всё же требует привязку порта даже для worker-сервиса,
задайте `PORT` — поднимется минимальный healthcheck-сервер, не влияющий на
логику бота. Полный разбор вариантов (включая почему Render/Fly.io/Railway не
подходят бесплатно для always-on воркера в 2026 году) — в DEPLOY.md.

## Firestore-индексы

Пара запросов (`/list`, `/excluded`) фильтруют по `status` и одновременно
сортируют по другому полю (`addedAt`/`excludedAt`) — Firestore требует для
этого составной индекс. При первом реальном вызове такого запроса Admin SDK
вернёт ошибку `FAILED_PRECONDITION` с прямой ссылкой на консоль Firebase для
одного клика создания нужного индекса — это ожидаемо, просто перейдите по
ссылке один раз на проде.

## Модель данных (Firestore)

- `suggestions/{id}` — предложения маршрутов (`seq`, `text`, статус active/excluded)
- `polls/{id}` — опросы (открыт/закрыт, соответствие опций и предложений, победитель)
- `config/main` — единственный документ конфигурации (chat id группы, расписание)
- `adminRegistrations/{userId}` — кто из пользователей нажал `/start` в ЛС
- `counters/suggestionSeq` — счётчик для присвоения `seq`
